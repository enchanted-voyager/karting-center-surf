# Фича 1: Просмотр расписания заездов — `GET /schedule`

## 1. Краткая цель фичи

Дать клиенту возможность увидеть список доступных заездов на сегодня: время старта, конфигурацию трассы и количество свободных карт, — чтобы он мог выбрать удобный заезд для записи.

## 2. Требования

Фича реализует:

- **User Story 1** («Просмотр расписания заездов») из `task1_requirements.md`: «Я, как клиент, хочу видеть расписание заездов на ближайшие дни с указанием времени старта, конфигурации трассы (короткая/длинная) и количества свободных карт, чтобы выбрать удобный заезд».
- **Сценарий 1** («Просмотр расписания и выбор заезда») из `task1_requirements.md`: пользователь открывает раздел «Расписание», видит для каждого заезда дату/время старта, конфигурацию трассы, имя маршала (только для отображения) и количество свободных карт.
- Требование из брифа заказчика (`brief-karting.md`): «Клиент должен видеть свободные слоты и записаться».

В моке эндпоинт возвращает данные из in-memory списка заездов (`Db.Slots`), включая ID заезда, время старта, конфигурацию трассы и текущее число свободных карт.

## 3. Ручная проверка

Код запущен в консоли (`dotnet run`) и проверен вручную: при выборе пункта меню «1. GET /schedule» приложение корректно выводит список заездов на сегодня с ID, временем старта, конфигурацией трассы и количеством свободных карт, включая заезд с 0 свободных мест.
