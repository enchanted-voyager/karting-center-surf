# Фича 3: Отмена брони — `DELETE /bookings/{id}`

## 1. Краткая цель фичи

Дать клиенту возможность отменить свою запись на заезд заранее, чтобы освободить место для других, при этом бронь удаляется из списка, а место возвращается в счётчик свободных карт заезда.

## 2. Требования

Фича реализует:

- **User Story 3** («Отмена бронирования») из `task1_requirements.md`: «Я, как клиент, хочу иметь возможность отменить свою запись заранее, чтобы освободить место для других».
- **Сценарий 2** («Бронирование заезда»), шаг 5, из `task1_requirements.md`: при отмене брони заранее (до заезда) бронь отменяется, слот освобождается.
- Требование из брифа заказчика (`brief-karting.md`) о том, что отмену записи нужно как-то ограничить, но не злить клиентов, — в текущем моке реализована базовая механика отмены и возврата места (без проверки временного окна «10 минут до старта», которая относится к полноценной реализации фичи).

В моке эндпоинт принимает ID брони, находит соответствующую запись в `Db.Bookings`, удаляет её из списка и увеличивает `FreeKarts` у связанного заезда на единицу.

## 3. Ручная проверка

Код запущен в консоли (`dotnet run`) и проверен вручную: при выборе пункта меню «3. DELETE /bookings/{id}» с указанием ID существующей брони приложение удаляет бронь из списка и возвращает свободное место на заезд (что подтверждается повторным вызовом «GET /schedule» — счётчик свободных карт увеличивается); при вводе несуществующего ID брони приложение корректно сообщает, что бронь не найдена.
