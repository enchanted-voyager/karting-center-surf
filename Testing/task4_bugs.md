# Тестирование и исправление багов

## Тест-кейсы

1. **GET /schedule** — расписание на сегодня выводится корректно: для каждого заезда показаны ID, время старта, конфигурация трассы и число свободных карт.
2. **POST /bookings** — бронирование заезда с корректным ID и типом экипировки уменьшает число свободных карт на 1 и создаёт запись с уникальным ID брони; попытка забронировать заезд без свободных мест должна отклоняться.
3. **DELETE /bookings/{id}** — отмена существующей брони удаляет её из списка и возвращает свободное место на заезд; попытка отменить несуществующую бронь должна возвращать понятное сообщение, а не падать с ошибкой.

## Баг №1: Возможность забронировать заезд, когда свободных мест уже 0

### Симптом

При выборе заезда с `FreeKarts = 0` (например, заезд ID 3 в тестовых данных) метод `CreateBooking` всё равно создавал бронь. Счётчик свободных карт уходил в отрицательное значение (`-1`), а бронь фактически создавалась поверх несуществующего места.

### Требования

Согласно **User Story 2** («Бронирование заезда») и **Сценарию 2** из `task1_requirements.md`, при отсутствии свободных мест приложение должно **отказывать** в бронировании и показывать понятное сообщение, а не создавать бронь. Это также напрямую связано с требованием заказчика из `brief-karting.md` о гарантии «0 двойных броней» — счётчик свободных карт не должен уходить в минус.

### Исправление

Причина — ошибка в граничном условии проверки: код пропускал бронирование, если `slot.FreeKarts < 0`, вместо `slot.FreeKarts <= 0`. Из-за этого случай «ровно 0 свободных мест» ошибочно считался допустимым для брони.

Было:
```csharp
if (slot.FreeKarts < 0)
{
    Console.WriteLine("Свободных мест на этот заезд нет.");
    return;
}
```

Стало:
```csharp
if (slot.FreeKarts <= 0)
{
    Console.WriteLine("Свободных мест на этот заезд нет.");
    return;
}
```

### Ручная проверка

Исправленный код запущен в консоли (`dotnet run`): при попытке забронировать заезд с ID 3 (0 свободных мест) приложение теперь корректно выводит сообщение «Свободных мест на этот заезд нет.» и не создаёт бронь. Баг больше не воспроизводится.
